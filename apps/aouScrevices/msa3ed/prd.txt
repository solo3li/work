=============================
        القسم الأول: تطبيقات العميل (Expo)
=============================

📱 التطبيقات:
- Expo App للمستخدم (Student)
- Expo App للمنفذ (Executor)
> يمكن يكون نفس التطبيق Multi Role أو تطبيقين منفصلين

=============================
        المستخدمين
=============================

👤 الطالب (Student)
- تسجيل حساب
- تسجيل دخول
- تأكيد OTP عبر الإيميل في كل دخول
- تصفح الخدمات
- البحث والفلترة
- إنشاء طلب خدمة
- رفع ملفات عند الطلب
- الدفع عبر Paymob
- متابعة حالة الطلب
- شات داخل الطلب
- شات مباشر مع المنفذ
- فتح تذكرة دعم
- تأكيد استلام الخدمة
- تقييم المنفذ لاحقًا (مرحلة بعد MVP)

👨‍💻 المنفذ (Executor)
- تسجيل حساب
- تسجيل دخول
- OTP في كل دخول
- رفع بيانات KYC
- انتظار الموافقة
- تصفح الطلبات المتاحة
- قبول الطلبات المسموح بها
- تنفيذ الطلبات
- رفع ملفات التسليم
- شات داخل الطلب
- شات مباشر
- فتح تذكرة دعم
- متابعة الأرباح والحالة

=============================
        ملاحظات هندسية مهمة
=============================

1- لا تعمل تطبيقين منفصلين .
الأذكى MVP = تطبيق Expo واحد Role Based.

2- لا تبدأ Microservices.
ابدأ Monolith نظيف.

3- افصل Admin Panel عن الموبايل بالصلاحيات فقط.

4- اجعل الدفع Webhook Driven وليس Client Driven.

5- ابدأ Local File Storage  

6- لو أهملت Logging / Audit Trails من البداية ستندم.
